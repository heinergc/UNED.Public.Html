Resumen Ejecutivo

El Sistema Robot de Sincronización de Active Directory implementa un sistema completo y automatizado de notificaciones por correo electrónico que consta de 5 tipos principales de reportes, diseñados para mantener informados a los analistas de TI y a los usuarios finales sobre el estado de las cuentas de Active Directory y el proceso de sincronización.

Este sistema garantiza la comunicación proactiva sobre vencimientos de cuentas, estadísticas del proceso y detección de cuentas deshabilitadas, permitiendo una gestión eficiente y oportuna del directorio activo institucional.

📋 Reportes Implementados

🚀

1. Notificación de Inicio de Proceso

Destinatario: Analista | Template: TemplateNotificacionInicio.html
Propósito:
Informar al analista que el proceso de sincronización ha iniciado correctamente
Contenido:
Fecha/hora de inicio, confirmación de ejecución, información del sistema
Configuración:
NotificarInicioProceso: True
📊

2. Notificación de Finalización con Estadísticas

Destinatario: Analista | Template: TemplateNotificacionFinalizacion.html
Propósito:
Reporte completo con estadísticas detalladas del proceso completado
Contenido:
  • Estadísticas completas: Procesados, errores, porcentajes de éxito
  • Estado de vigencia: Cuentas expiradas, por expirar y vigentes
  • Tabla detallada con funcionarios con incidencias
  • Datos de auditoría: Usuario, IP, equipo, sistema
  • Archivos LOG comprimidos en ZIP adjuntos
Características:
Gráficos visuales Archivos adjuntos
👤

3. Notificación Individual de Vigencia de Cuenta

Destinatarios: Usuarios finales | Template: TemplateNotificacionIndividualCuenta.html
Propósito:
Notificar a cada funcionario que su cuenta AD está próxima a expirar
Personalización:
  • Nombre completo del funcionario
  • Usuario de Active Directory
  • Fecha exacta de expiración y días restantes
  • Nivel de urgencia dinámico: URGENTE (≤7 días) | IMPORTANTE (>7 días)
  • Instrucciones detalladas para renovar contraseña
  • Enlaces a portales de autoservicio
Criterio de envío:
Cuentas que expiran en ≤ 30 días (configurable)
Configuración:
NotificarUsuariosIndividualmente: True DiasAvisoCuentaExpira: 30
📈

4. Reporte Consolidado de Vigencia de Cuentas

Destinatario: Analista | Template: TemplateConsolidadoVigencia.html
Propósito:
Resumen consolidado de todos los funcionarios con problemas de vigencia
Contenido:
  • Estadísticas: Total afectados, expiradas, por expirar
  • Tabla detallada con identificación, nombre, usuario AD, estado
  • Fecha de expiración y días restantes de cada cuenta
  • Códigos de color por nivel de criticidad
Configuración:
EnviarConsolidadoVigencia: True
🔒

5. Reporte de Cuentas Deshabilitadas

Destinatario: Analista | Template: TemplateReporteCuentasDeshabilitadas.html
Propósito:
Informar sobre cuentas AD deshabilitadas excluidas del proceso de sincronización
Contenido:
  • Total de cuentas deshabilitadas detectadas
  • Tabla con: Identificación, nombre, usuario AD, programa, estado
  • Mensaje de acción recomendada (si hay >10 cuentas)
  • Template inline de respaldo para mayor robustez
Configuración:
NotificarCuentasDeshabilitadas: False (por defecto)
